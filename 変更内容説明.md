# 夕食進行ボード - コンパクト化の変更内容

## 📝 変更の目的
タブレット縦画面でより多くの情報を表示できるように、画面の高さをコンパクトにしました。

---

## ✨ 主な変更点

### 1️⃣ 上部ボタンの配置変更（index.html）

**変更前:**
```
[本日の設定ページへ] [オーダー入力] [キッチン表示]  ← 横に3つ並んでいた
```

**変更後:**
```
[本日の設定ページへ]  ← 単独で上に配置
[オーダー入力] [キッチン表示]  ← タブとして横並び
```

**理由:**
- 参考画像に合わせて縦方向のスペースを節約
- 「本日の設定ページへ」は別のページへのリンクなので、独立して配置
- タブ切替ボタンは使用頻度が高いので、そのまま横並びで維持

---

### 2️⃣ 料理名ヘッダー行の削除（style.css）

**変更前:**
```
┌─────────────────────────────────────────┐
│ 部屋/スピード │ 吸物 │ 刺身 │ 蒸物 │ ... │  ← この行があった
├─────────────────────────────────────────┤
│ やまぶき      │  ○  │  ○  │  ○  │ ... │
└─────────────────────────────────────────┘
```

**変更後:**
```
┌─────────────────────────────────────────┐
│ やまぶき      │  ○  │  ○  │  ○  │ ... │  ← ヘッダー無し
└─────────────────────────────────────────┘
```

**追加されたCSSルール:**
```css
.thead {
  display: none !important;
}
```

**理由:**
- 各料理名は丸ボタンの上に表示されているため、ヘッダー行は重複
- ヘッダー行を削除することで約40px程度の高さを節約
- 視覚的にもスッキリして見やすくなる

---

### 3️⃣ 各要素の余白削減（style.css）

#### 📌 セル内の余白
```css
/* 変更前 */
.th, .td {
  padding: 8px;
}

/* 変更後 */
.th, .td {
  padding: 6px 4px;  /* 上下6px、左右4px */
}
```

#### 📌 料理名の高さ
```css
/* 変更前 */
.dishname {
  font-size: 12px;
  min-height: 18px;
}

/* 変更後 */
.dishname {
  font-size: 11px;
  min-height: 16px;
}
```

#### 📌 時間グループの余白
```css
/* 変更前 */
.time-group {
  margin-bottom: 16px;
}
.time-group-header {
  margin: 8px 0 12px 0;
}

/* 変更後 */
.time-group {
  margin-bottom: 12px;
}
.time-group-header {
  margin: 6px 0 10px 0;
}
```

**理由:**
- 文字サイズは読みやすさを保ちながら最小限に調整
- 余白を詰めることで、1画面に表示できる部屋の数が増える
- タブレット縦画面でスクロールが減る

---

## 📊 削減された高さの概算

| 項目 | 削減量 |
|------|--------|
| 料理名ヘッダー行の削除 | 約40px |
| セル内余白の削減 (1行あたり) | 約4px |
| 料理名の高さ削減 (1セルあたり) | 約2px |
| グループ間余白の削減 | 約12px (3グループで) |
| **合計** | **約60～80px** |

※ 10部屋表示の場合、1画面に表示できる情報量が約15～20%増加

## 🔧 変更されたファイル

1. **index.html** - ボタン配置の変更
2. **style.css** - ヘッダー削除と余白調整
3. **app.js** - 変更なし（そのまま使用可能）
4. **settings.html** - 変更なし（そのまま使用可能）

## ✅ 動作確認のポイント

1. 上部のボタンが縦に並んでいるか
2. 料理名のヘッダー行（吸物、刺身、蒸物...）が表示されていないか
3. 各部屋の行が以前より詰まって表示されているか
4. 丸ボタンのクリック機能が正常に動作するか
5. タブレット縦画面で、より多くの部屋が1画面に収まっているか

---

## 💡 今後の調整について

もし「もう少し余白が欲しい」「逆にもっと詰めたい」などのご要望があれば、style.cssの以下の値を調整してください：

```css
/* セル内の余白 */
.th, .td {
  padding: 6px 4px;  /* この数値を変更 */
}

/* 料理名のサイズ */
.dishname {
  font-size: 11px;  /* この数値を変更 */
}

/* グループ間の余白 */
.time-group {
  margin-bottom: 12px;  /* この数値を変更 */
}
```
